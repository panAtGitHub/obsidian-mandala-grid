# 第二轮修正计划：补齐 `nx9` Draft Projection、提交语义与测试覆盖

## 0. 给下一个 AI 的执行说明

这份计划不是重新推翻上一轮实现，而是在上一轮已经加入的 `edit-session` 基础上补齐未落地的部分。

核心目标只有一个：桌面端主区 inline 编辑和右侧详情栏编辑时，按键输入不能再穿透完整 `document -> scene -> nx9 runtime -> side effects` 链路。每次按键只更新编辑器本地 buffer；短 debounce 后只局部更新当前正在编辑的 cell；只有 idle、blur、save、切换 node、unload、退出编辑时才正式 commit 到领域层。

实施时必须优先保持补丁小而可验证。不要顺手重构无关模块，不要改 mobile/native editor，不要把 draft 写入 `documentState`。

## 2. 总目标和非目标

### 2.1 总目标

实现三层编辑模型：

1. 编辑缓冲层：每次按键只更新编辑器本地 buffer 和 `EditSessionService.bufferContent`。
2. 投影草稿层：150ms trailing debounce 后，只把当前正在编辑 node 的 `{ nodeId, content, revision }` 投影到 scene/root/nx9，触发当前 cell 的 content-only patch。
3. 领域聚合层：600ms idle debounce 或 blur/save/switch/unload/disable-edit 时，才把内容 commit 到 `documentStore`。

最终效果：

- 右侧详情栏打开时，在 `nx9` 中输入不再导致整页 runtime 重跑。
- draft 只影响当前编辑 node 对应 cell。
- 非当前 node 的 cell、row、section 引用保持稳定。
- `documentState` 只保存 committed content。
- save/search/align/focus/persist 等重副作用只在 commit 后发生。
- blur commit 保存内容，但不能把焦点重新抢回 editor。

### 2.2 非目标

本轮不做以下事情：

- 不改 mobile/native editor session。
- 不重构完整 document reducer 架构。
- 不改 pagination、topology、section facts 的生成逻辑。
- 不把 draft 写入 `documentState`。
- 不引入多 session；桌面端同一时刻仍只允许一个 active edit session。
- 不删除现有 `contentStore` overlay，除非确认所有非 scene 路径都有替代方案。

### 阶段 1：收口 `edit-session` API 和提交语义

修改文件：

- `src/view/edit-session/edit-session-types.ts`
- `src/view/edit-session/edit-session-service.ts`

新增或确认这些类型：

```ts
export type EditSessionCommitReason =
    | 'idle'
    | 'save'
    | 'blur'
    | 'switch-node'
    | 'unload'
    | 'disable-edit';

export interface EditSessionCommitPayload {
    nodeId: string;
    content: string;
    isInSidebar: boolean;
    reason: EditSessionCommitReason;
    suppressRefocus: boolean;
}

export interface EditSessionProjectionSnapshot {
    nodeId: string;
    content: string;
    revision: number;
}
```

`EditSessionService` 必须提供这些语义清晰的方法：

```ts
startSession(nodeId: string, isInSidebar: boolean, initialContent: string): void;
updateBuffer(content: string): void;
flushProjection(): void;
commit(reason: EditSessionCommitReason): void;
requestSave(): void;
requestBlurCommit(): void;
switchNode(nextNodeId: string, isInSidebar: boolean, initialContent: string): void;
cancel(): void;
endSession(reason: EditSessionCommitReason): void;
getProjectionSnapshot(): EditSessionProjectionSnapshot | null;
```

行为要求：

- `updateBuffer(content)` 只更新 buffer、dirty、timer，不 dispatch document action。
- projection timer 固定 150ms trailing debounce。
- commit timer 固定 600ms trailing debounce，reason 为 `idle`。
- `requestSave()` 立即 `commit('save')`。
- `requestBlurCommit()` 立即 `commit('blur')`，但不结束 session。
- `switchNode(...)` 如果旧 session dirty，先 `commit('switch-node')`，再启动新 session。
- `cancel()` 丢弃 buffer/projection，不 dispatch。
- `endSession('disable-edit')` 或 `endSession('unload')` 在 dirty 时先 commit，再清空 session。
- dirty=false 时 commit 不 dispatch，不增加 revision。
- commit 后必须清理 commit timer。
- projection commit 后可以保留或清空，但最终不能让旧 node 的 projection 污染新 node。建议切换 node 或 end session 时清空 projection。

`onCommit` 回调 payload 必须带 `reason` 和 `suppressRefocus`：

```ts
suppressRefocus: reason === 'blur'
```

### 阶段 6：实现 `nx9` 单 cell content-only patch

修改文件：

- `src/mandala-scenes/view-nx9/page-runtime.ts`
- `src/mandala-scenes/view-nx9/assemble-cell-view-model.ts`

新增 runtime 入参：

```ts

```

runtime cache entry 增加：

```ts
draftProjection: SceneDraftProjectionSnapshot | null;
```

判断 draft 是否变化：

```ts
function sameDraftProjection(
    a: SceneDraftProjectionSnapshot | null,
    b: SceneDraftProjectionSnapshot | null,
): boolean {
    return (
        a?.nodeId === b?.nodeId &&
        a?.revision === b?.revision &&
        a?.content === b?.content
    );
}
```

建议 patch 算法：

1. 如果 static rows、page frame、page index、interaction snapshot 都与上一帧兼容，并且只有 draftProjection 变化，则走 draft patch。
2. 计算受影响 node id 集合：旧 draft node 和新 draft node。
3. 如果受影响 node 都不在当前 page，直接返回上一帧 rows 引用。
4. 如果受影响 node 在当前 page，只复制对应 row 和对应 cell。
5. 重建该 cell 的 card view model，并传入 `contentOverride`。
6. 其他 row/cell/card view model 保持原引用。
7. trace 标记为 draft patch，不能标记为 full apply。

伪代码：

```ts
function patchNx9DraftProjectionState(args: {
    rows: Nx9RuntimeRow[];
    staticRows: Nx9StaticRow[];
    pageIndex: Nx9PageIndex;
    previousDraftProjection: SceneDraftProjectionSnapshot | null;
    nextDraftProjection: SceneDraftProjectionSnapshot | null;
}): Nx9RuntimeRow[] {
    const affectedNodeIds = new Set(
        [args.previousDraftProjection?.nodeId, args.nextDraftProjection?.nodeId]
            .filter((nodeId): nodeId is string => Boolean(nodeId)),
    );

    if (affectedNodeIds.size === 0) {
        return args.rows;
    }

    let changed = false;

    const nextRows = args.rows.map((row) => {
        const cellIndex = row.cells.findIndex((cell) =>
            affectedNodeIds.has(cell.nodeId),
        );

        if (cellIndex < 0) {
            return row;
        }

        const nextCells = row.cells.slice();
        nextCells[cellIndex] = rebuildCellWithDraftProjection(
            nextCells[cellIndex],
            args.nextDraftProjection,
        );
        changed = true;

        return {
            ...row,
            cells: nextCells,
        };
    });

    return changed ? nextRows : args.rows;
}
```

实际实现应按当前 `Nx9RuntimeRow` / cell 类型调整，不要为了匹配伪代码改大范围类型。

必须满足：

- 当前编辑 node 的 draft 变化只 patch 当前 cell。
- 非当前 node 的 cell 引用不变。
- 非当前 row 引用不变。
- 不重建 `pageFrame`。
- 不重建 `staticRows`。
- 不重建 `structure context`。
- 不触发热页之外 page cache 变化。
- 不让整页 `rows` 每次输入整体重算。

- 不要同时让同一个 nx9 cell 既通过 `contentOverride` 又通过 `contentStore` overlay 重复订阅，避免双重更新。

如果 `Content.svelte` 已拿到 `contentOverride`，它应优先渲染 override，并且这条路径不需要订阅 `contentStore` 的 projection overlay。

### 阶段 9：修复副作用边界和 blur 抢焦点

修改文件：

- `src/stores/view/subscriptions/on-document-state-update.ts`
- `src/stores/view/subscriptions/effects/focus-container.ts`
- 相关 spec。

目标：

- projection 更新不触发任何副作用。
- commit 后触发 save/search/align/persist 等已有副作用。
- `action.context.suppressRefocus === true` 时跳过 `focusContainer(view)`。
- blur commit 保存内容，但不把焦点拉回 inline editor。

建议实现：

```ts
const shouldRefocus = action.context?.suppressRefocus !== true;

if (shouldRefocus) {
    focusContainer(view);
}
```

如果 `focusContainer(...)` 被多个地方直接调用，优先在 `on-document-state-update.ts` 过滤；必要时再给 `focusContainer(view, { suppressInlineEditorFocus: true })` 加 options。

必须保持：

- `idle` commit 后现有行为尽量不变。
- `save` commit 后现有保存链路不变。
- `disable-edit` / `unload` commit 后现有退出编辑行为不变。
- mobile 逻辑不变。

## 7. 交付检查清单

代码完成后必须逐项确认：

## 8. 禁止事项

不要做这些事：

- 不要把 draft 写进 `documentState.document.content`。
- 不要让 `projectionStore` dispatch `document/update-node-content`。
- 不要在 `Content.svelte` 里直接读取 `EditSessionService`。
- 不要把 `draftProjection` 放进 topology、section facts、structure context 的 cache key。
- 不要为了 draft 更新重建整页 `rows`。
- 不要让 blur commit 后调用 `view.inlineEditor.focus()`。
- 不要让主区编辑和右侧详情栏编辑走两套不同规则。
- 不要改 mobile/native editor。
- 不要大范围重排 imports 或格式化全仓库。

## 9. 最终成功标准

本轮实现成功的标准不是“看起来能输入”，而是这些可验证条件同时成立：

- 每次按键只更新本地 buffer。
- 150ms 后只有当前编辑 node 的 projection 更新。
- `nx9` 当前 page 中最多当前 cell 被替换。
- 非当前 row/cell 引用稳定。
- 600ms idle 或 blur/save/switch/unload 后才 commit 到 document store。
- `contentRevision` 只在 commit 时增加。
- projection 更新不会触发 save/search/align/focus。
- blur commit 不抢焦点。
- 右侧详情栏打开时连续输入比当前实现更顺，且其他 section 在输入中保持静止。
