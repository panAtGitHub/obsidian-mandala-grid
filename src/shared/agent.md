# src/shared/agent.md

说明 `src/shared` 目录的职责，方便后续判断哪些代码是真正的跨模块通用能力。

## 总体职责

- `src/shared/`
  负责不带 mandala 业务语义的通用能力。
- 只有真正可跨模块复用、且不依赖具体场景/格子/设置语义的代码，才应该放这里。
- 如果某段代码明显带有 mandala 业务词汇，优先考虑放回对应业务模块，而不是 `shared`。

---

## 当前目录含义

- `helpers/`
  放通用辅助函数。

- `store/`
  放通用 store 实现与 store 辅助。

- `lib/`
  放可复用的基础库辅助。
  当前包括对齐、错误、格式检测等。

- `ui/`
  放通用 UI 原子组件或共享 UI 片段。

- `test-helpers/`
  放测试辅助代码。

---

## 与其他模块的边界

- `src/shared/`
  负责“跨模块通用能力”。

- `src/mandala-*`
  负责具体业务能力。

判断标准：

- 如果一个函数删掉 “mandala” 语境仍然成立，才适合放 `shared`。
- 如果一个组件只服务某个业务模块，不要放 `shared/ui`。

---

## 修改建议

- 要改纯工具函数、通用 store、基础 UI，优先看这里。
- 新代码进 `shared` 前，先确认它是否真的不依赖业务语义。
- 发现 `shared` 里出现过多 mandala 专属词汇时，应考虑回迁到具体业务模块。
