# 2026-03-21 ～ 2026-03-22（架构升级记录）

## 本次升级重点

- 核心目录完成重组：原先分散的视图、格子、设置、文档与显示逻辑，分别收口到 `mandala-scenes`、`mandala-cell`、`mandala-settings`、`mandala-document`、`mandala-display` 等核心模块中，减少旧 `view/` 目录承担过多职责的问题。
- 各场景完成独立拆分：`3x3`、`9x9`、`nx9`、`7x9` 逐步拆成各自的 `layout + assemble-cell-view-model` 结构，场景文件开始只负责布局与组装，减少把格子内部实现直接写在场景里的情况。
- 格子模块完成下沉：卡片内容、meta、交互、滚动条、render model、view model 等逻辑持续回收到 `mandala-cell`，让格子本身成为相对独立的“默认能力仓库”。
- 设置与共享逻辑持续瘦身：导出、模板、设置动作、移动端编辑器视口、文档订阅副作用、部分 sidebar / view 运行时逻辑被陆续抽离，减少超大总控文件的耦合度。

## 场景与格子的边界调整

- `3x3 / nx9 / 7x9` 原先直接写在 scene 中的 `.lng-prev`、`.editor-container`、`.cm-scroller` 等格子内部样式，已逐步回收到 `mandala-cell`，scene 不再直接操作格子内部 DOM 细节。
- `3x3` 先完成了“场景微调外置”示范：格子默认展示配置由 `mandala-cell` 提供，`view-3x3` 单独维护 `build-cell-display-overrides.ts`，只表达本场景相对默认格子的微调项。
- `nx9 / 7x9` 也同步切换到同样的组织方式：不再由 `cell-display-policy.ts` 按场景名统一分发，而是由各场景自己的 assembler 合并“默认配置 + 场景 overrides”。
- `cell-display-policy.ts` 现在只保留“格子显示配置”的类型定义；`default-cell-display-policy.ts` 负责标准默认值；各 scene 自己维护微调文件。这让“默认值”和“场景差异”开始分层清楚。

## 视觉与交互相关修正

- 格子正文滚动条的实现被收口到 `mandala-cell`：滚动条样式、滚动行为与挂载入口不再散落在多个场景文件中，便于后续集中调整。
- 针对进入编辑态再退出后，格子内部出现横向滚动条的问题，开始从 `cell` 内部盒模型与滚动容器约束入手修正，而不是继续由场景外层兜底。
- 自定义 section 背景色的输入语义被统一：当存在用户自定义颜色时，显示层统一以“用户色 + 透明度”作为背景输入来源，减少标准卡片与 `9x9` 摘要格各自解释背景色的情况。

## 文档与阅读体验补充

- 为核心目录补充了 `agent.md`，明确各模块的职责边界，特别是 `mandala-cell` 与 `mandala-scenes` 的分工：`cell` 负责默认格子能力，`scene` 负责布局与局部微调。
- 为 `mandala-cell/model` 增补了“阅读顺序”说明，帮助从类型定义、默认值、场景微调、view model、render model 这条链路理解格子模块，降低新读代码时的理解门槛。

## 当前阶段结论

- 这两天的升级，不是单点功能新增，而是一次较大规模的内部结构整理。
- 当前最核心的变化是：场景不再直接控制格子内部实现，格子逐步成为统一的默认呈现层；场景只在自己的目录里表达“本场景相对默认格子的微调”。
